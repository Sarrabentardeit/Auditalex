Alexann – Audit d'Hygiène (PWA)

Documentation technique – Architecture, paramétrages et maintenance

1. Architecture technique du projet

Alexann – Audit d'Hygiène repose sur une architecture client-serveur avec base de données NoSQL.

1.1 Vue d'ensemble

1.2 Frontend (application web)

  • Technologies : React 19, TypeScript, Vite 7, Material-UI (MUI v7), React Router, Zustand
  • Fonctionnalités : génération PDF (jsPDF), compression d'images (browser-image-compression), PWA (Workbox)
  • Structure : pages (Login, Dashboard, Audit, ActionsCorrectives, Users), composants réutilisables, store Zustand pour l'état global
  • Données : grille d'audit définie dans data_structure.json (catégories, items, observations, mesures correctives)

1.3 Backend (API REST)

  • Technologies : Node.js, Express, TypeScript, Mongoose
  • Sécurité : JWT, bcrypt, Helmet, CORS, rate limiting, compression
  • Routes principales :
    • /api/auth : connexion, vérification de session
    • /api/users : gestion des utilisateurs (admin)
    • /api/audits : CRUD des audits
  • Base de données : MongoDB (Atlas ou instance locale Docker)

1.4 Infrastructure de déploiement

  • Développement : frontend (Vite, port 5173), backend (port 3000), MongoDB (port 27017 ou Atlas)
  • Production (Docker) : 3 conteneurs (frontend Nginx, backend Node, MongoDB)
  • VPS : Nginx en reverse proxy, SSL (Let's Encrypt), redirection HTTP → HTTPS
  • URL : alexann.nav.ovh (ou domaine configuré)

3. Points d'attention pour la maintenance

3.1 Base de données (MongoDB)

  • Sauvegardes : planifier des sauvegardes régulières (quotidiennes ou hebdomadaires)
  • Taille : les audits avec photos (base64) peuvent augmenter fortement le volume ; surveiller l'espace disque
  • Index : index sur createdAt et auditorId pour les requêtes de liste ; vérifier les performances si la base grossit

3.2 Sécurité

  • JWT_SECRET : utiliser une valeur longue et aléatoire, ne jamais la committer dans le dépôt
  • Mots de passe MongoDB : stocker uniquement dans des variables d'environnement ou un gestionnaire de secrets
  • Renouvellement SSL : Certbot/Let's Encrypt ; vérifier que le renouvellement automatique fonctionne

3.3 Performance

  • Chargement des audits : les audits avec beaucoup de photos sont chargés sans photos par défaut, puis les photos sont chargées en arrière-plan
  • Limite JSON : le backend accepte des requêtes jusqu'à 100 MB pour les audits avec photos
  • Nginx : client_max_body_size 50M ; augmenter si nécessaire pour des uploads plus lourds

3.4 Maintenance applicative

  • Mises à jour : mettre à jour les dépendances npm régulièrement (sécurité et correctifs)
  • Logs : consulter les logs Nginx (/var/log/nginx/audit_*.log) et les logs backend en cas d'incident
  • Scripts utilitaires : init-admin pour créer un admin, cleanup-audits pour nettoyer les doublons éventuels

3.5 Surveillance

  • Health check : GET /health et GET /api/health pour vérifier que le backend répond
  • Docker : healthchecks configurés sur les conteneurs ; surveiller les redémarrages
  • MongoDB : surveiller la connexion et les erreurs de requêtes

3.6 Évolutions prévues

  • Intégration Nextcloud : envoi des PDF vers un serveur Nextcloud (WebDAV) ; à paramétrer selon les besoins du client
  • Grille d'audit : toute modification de data_structure.json nécessite un redéploiement du frontend
